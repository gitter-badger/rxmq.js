    EndlessSubject

    Extends:

    rx~Rx.Subject → EndlessSubject

    EndlessSubject extension of Rx.Subject. This is pretty hacky, but so far I'd found no better way of having Subjects that do no close on multicasted stream completion. For documentation refer to Rx.Subject docs. The only difference is that EndlessSubject never triggers '.onCompleted()'
